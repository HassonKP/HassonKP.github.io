About the Game

This SimpleShooter game is a fairly basic and straight forward 3rd person shooter, again done in Unreal Engine. The Map itself was pre-made and the Player/NPC assets themselves come from the Paragon:Wraith pack on the Unreal Store. The game has you the player move through a command center and eliminate hostiles, the AI has some basic features in that if you are spotted they will chase and fire when within range. The win condition is simple, destroy all enemy targets before you find yourself at 0 hit points.

Why I Did This Project

This project was the final one in the Udemy course, and to be fair probably my favorite project. It took the fundamentals that were laid out in ToonTanks and really expanded on them. It was the first time implementing true animations in Unreal, and introduced everything from Blend Spaces to Animation Blue Prints. This course expanded upon the use of widgets allowing for, albeit basic, a actual HUD that was similar enough to so many games I had enjoyed before.

One of the most exciting aspects of this project was creating a more engaging AI enemy character, moving beyond the static Towers in ToonTanks that merely aimed and fired at the player. I explored Behavior Trees and Blackboards, which enabled me to design enemies capable of perceiving and reacting to the player. In this aspect I felt like I was understanding finally how the world itself could perceive and interact with the Player in order to create a more interesting game play loop.

This project took the concepts and fundamentals that had been laid out in earlier courses and allowed me to create a game close to something I could play.

What I Learned

  • Blend Spaces
    • Used to smoothly transition between animations (idle to running, and direction).
  • Animation Blueprints
    • Centralized control over character animations using state machines.
    • Enabled dynamic interactions between character movement and animation.
  • Behavior Trees
    • Designed complex enemy AI decision-making systems.
    • Implemented key components such as:
      • BTTasks: Custom logic nodes for specific enemy actions, such as attacking or patrolling.
      • BTServices: Periodic checks for player proximity or health, adding dynamic reactions to AI.
  • Blackboard Values
    • Managed AI states, such as awareness of the player or patrolling paths, and ability to fire.
    • Linked to Behavior Trees for real-time updates.
  • Custom C++ Gun/Weapons and Blueprints
    • Developed modular weapon systems to handle shooting mechanics.
    • Integrated with Blueprints for ease of use and rapid iteration.
  • Custom AI Controllers
    • Controlled AI behavior, delegating tasks and making decisions based on environmental factors.
    • Allowed modularity by separating AI logic from character data.
  • Player Controllers
    • Managed player input and interaction with the game world.
  • HUD and Widgets
    • Built a minimalistic yet functional Heads-Up Display (HUD) to show health and crosshair.
  • Sound Design
    • Incorporated sound effects for firing, hitting targets, and other in-game events.
  • Dynamic Mesh Manipulation
    • Programmatically controlled parts of the character.
  • Game Flow Control
    • Implemented custom logic for starting, pausing, and ending the game.
